# Example — 前海现货竞买竞卖（QME）走查

一个真实跑通的例子，用来对照"做对了长什么样"。项目前缀 `QME`。

## 输入与产物

- 输入：1 份业务细则 PDF（10 页）；多份外部办法/公告**未提供**（已登记为外部依赖）。
- A 证据：47 条（`QME-E-001..047`，含页码、原文、SHA256、text_hash）。
- B 需求卡：41 张（`QME-REQ-001..041`），含 ambiguity 与 confidence。
- C 设计：按需求类型选技术；资金/担保/权限/合规升 P0。
- D 用例：66 条 L1 规则测试点（`QME-TC-001..066`）。因无系统输入资产，全部停在 L1。
- E 验证：Traceability 100%、文档需求引用率 100%、Confirmed 45 / Needs Confirmation 21、Rejected 模式 5 类（页面按钮名、担保金额、手续费率、结算细节、罚则——均因 PDF 无依据而拒绝臆造）。

## 两层模型在这里怎么用

- 加价竞买规则 `QME-REQ-008` 的 L1：
  ```
  预期：加价竞买应以初始价为起点，由低至高按最小加价幅度整倍数加价回应。
  ```
- 一旦拿到报价表单字段（初始价/最小加价幅度/最大加价幅度），注入为 L2：
  ```
  数据：105 接受；103 拒绝（非整倍数）；121 拒绝（>初始价+最大幅度）
  ```
  L2 粒度对齐既有库（"报价值 ≤ 初始价+最大价格变动幅度，增量为最小变动幅度整数倍"）。

## 对标（应做但首次缺失）

既有 CTCS 竞价模块有约 1,363 条真实用例。应在可比子集上算 Recall/Novelty/Precision——这是给管理层的核心证据。66 条 L1 vs 1,363 条具体用例，也直观说明"仅靠业务细则只能触及系统测试面的一小片"。

## 复盘要点（被本 skill 固化为闸门）

1. L1 不可执行是输入不足，不是模型不行 → 两层模型 + 输入注入。
2. "100% 覆盖"易误导 → 指标分层 + 汇报红线。
3. 32% NC 不应计入产物 → 输入充分性闸门 + NC 生命周期。
4. 出现 QME/QH 双 ID、多份重叠 Excel → 单一命名空间 + 单一权威产物。
5. 手握 1,363 条人工用例却没对标 → 金标准对标必做。
